# Root Folder Cleanup Plan

## Files to Remove

### 1. Legacy Configuration Files (Safe to Remove)
- `.eslintrc.json` - Replaced by Biome linter
- `.prettierrc` - Replaced by Biome formatter
- `pnpm-workspace.yaml` - Project uses Bun, not pnpm

### 2. Temporary/Test Files (Safe to Remove)
- `take-screenshot.js` - Playwright screenshot test script
- `test-navigation.js` - Navigation delay test script
- `test-image.txt` - Upload functionality test file
- `test-screenshot.png` - Test screenshot output
- `tmp_layout_before.txt` - Temporary layout backup
- `typography-test-screenshot.png` - Typography test screenshot
- `vibedevid_home.html` - Exported HTML snapshot

### 3. Build/Cache Files (Can be Regenerated)
- `tsconfig.tsbuildinfo` - TypeScript incremental build cache

### 4. Test Artifacts (Safe to Remove)
- `testsprite_tests/` - Testing framework artifacts directory

## Files to Keep (Do Not Remove)
- `.mcp.json` - Active MCP configuration for shadcn
- `tsconfig.tsbuildinfo` - If user prefers to keep build cache (optional)
- All project folders (`app/`, `components/`, `lib/`, etc.)
- All essential config files (`package.json`, `next.config.mjs`, etc.)
- All documentation files (`AGENTS.md`, `README.md`, etc.)
- Build folders (`.next/`, `node_modules/`) - These are typically gitignored

## Verification Steps
After removal:
1. Run `bun tsc --noEmit` to verify TypeScript still compiles
2. Run `bun lint` to verify Biome still works
3. Run `bun dev` to verify application starts correctly
4. Check that all tests still pass: `bunx playwright test`

## Rollback
If any issues occur, the files can be restored from git history:
```bash
git restore <filename>
```
or restore all removed files:
```bash
git restore .
```
