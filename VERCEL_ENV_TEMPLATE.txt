# ENVIRONMENT VARIABLES FOR VERCEL DEPLOYMENT
# Go to Vercel Dashboard > Your Project > Settings > Environment Variables

# Required environment variables:
# Copy these variable names and add your actual values in Vercel

VITE_SUPABASE_URL                 # Public Supabase project URL (https://xxxx.supabase.co)
VITE_SUPABASE_ANON_KEY            # Public Supabase anonymous key
VITE_SITE_URL                     # Production URL (e.g., https://vibedev.id)
SUPABASE_SERVICE_ROLE_KEY         # Supabase service role key (server-only secret)
UPLOADTHING_TOKEN                 # UploadThing API token (server-only secret)

# Optional environment variables:
OPENROUTER_API_KEY                # OpenRouter API key for AI features
GITHUB_TOKEN                      # Optional GitHub import token (or use GH_TOKEN)
GH_TOKEN                          # Optional GitHub import token alternative
NEXT_PUBLIC_SUPABASE_URL          # Legacy alias; keep aligned with VITE_SUPABASE_URL while app/ routes remain
NEXT_PUBLIC_SUPABASE_ANON_KEY     # Legacy alias; keep aligned with VITE_SUPABASE_ANON_KEY while app/ routes remain
NEXT_PUBLIC_SITE_URL              # Legacy alias; keep aligned with VITE_SITE_URL while app/ routes remain
NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL  # Public Supabase auth redirect URL

# IMPORTANT NOTES:
# 1. VITE_SUPABASE_URL must be a valid URL starting with https://
# 2. Do NOT use placeholder values like "sb_publishable_..." for the URL
# 3. Get your actual values from:
#    - Supabase: Project Settings > API
#    - UploadThing: Dashboard > API Keys
#    - OpenRouter: https://openrouter.ai/keys
# 4. Keep SUPABASE_SERVICE_ROLE_KEY, UPLOADTHING_TOKEN, OPENROUTER_API_KEY, and GitHub tokens private
# 5. `.env.example` is the canonical template for local and production env setup
