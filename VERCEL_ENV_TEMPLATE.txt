# ENVIRONMENT VARIABLES FOR VERCEL DEPLOYMENT
# Go to Vercel Dashboard > Your Project > Settings > Environment Variables

# Required environment variables:
# Copy these variable names and add your actual values in Vercel

NEXT_PUBLIC_SUPABASE_URL          # Your Supabase project URL (https://xxxx.supabase.co)
NEXT_PUBLIC_SUPABASE_ANON_KEY     # Your Supabase anonymous key
UPLOADTHING_TOKEN                 # Your UploadThing API token

# Optional environment variables:
NEXT_PUBLIC_SITE_URL              # Your production URL (e.g., https://vibedev.id)
NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL  # OAuth redirect URL
OPENROUTER_API_KEY                # OpenRouter API key for AI features (get from openrouter.ai)

# IMPORTANT NOTES:
# 1. NEXT_PUBLIC_SUPABASE_URL must be a valid URL starting with https://
# 2. Do NOT use placeholder values like "sb_publishable_..." for the URL
# 3. Get your actual values from:
#    - Supabase: Project Settings > API
#    - UploadThing: Dashboard > API Keys
#    - OpenRouter: https://openrouter.ai/keys
# 4. Keep UPLOADTHING_TOKEN, SUPABASE_SERVICE_ROLE_KEY, and OPENROUTER_API_KEY private (don't expose in client code)
